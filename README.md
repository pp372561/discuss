# Discuss2 - 轻量级论坛讨论系统

*基于 Spring MVC + JSP 开发的后端论坛项目，聚焦核心互动功能，适配本地开发与服务器快速部署，适合 Java Web 初学者入门实践。*

## 一、项目概述

### 1.1 项目定位

一款轻量、易扩展的基础论坛系统，覆盖用户管理、帖子互动全流程，可作为 Spring MVC 技术栈的实战案例，也可基于此进行二次开发。

### 1.2 核心特性

- 功能完整：包含用户登录注册、帖子发布/回复、权限控制等核心模块

- 易部署：支持 Maven 一键打包，服务器 Tomcat 直接部署

- 易上手：代码结构清晰，注释规范，适合初学者学习参考

## 二、功能详情

### 2.1 用户管理模块

- 账号注册：支持账号、密码、邮箱信息提交，前后端双重数据校验（非空、邮箱格式合法性）

- 登录登出：基于 Session 管理登录状态，验证通过后绑定用户身份，登出时销毁会话

- 权限控制：未登录用户仅可浏览内容，禁止发布、回复帖子等操作

### 2.2 帖子互动模块

- 帖子列表：展示所有帖子的标题、发布人、发布时间、回复数，支持快速预览核心信息

- 帖子发布：登录用户可提交标题+正文内容，自动关联发布用户与时间戳

- 详情查看：点击帖子标题进入详情页，展示完整内容及所有回复记录

- 多级回复：支持对帖子主体、已有回复进行二次互动，保留回复层级关系，提升交互体验

### 2.3 基础能力支撑

- 数据校验：前端 JS 预校验减少无效请求，后端通过 Hibernate Validator 强化校验逻辑

- 页面渲染：基于 JSTL 实现动态数据展示，路由跳转流畅，无冗余请求

- 异常处理：对非法操作（未登录发布、空内容提交等）给出友好提示，提升用户体验

## 四、快速运行指南

### 4.1 本地开发运行

1. 克隆项目到本地
        `git clone https://github.com/你的用户名/discuss2.git`

2. 使用 IDEA 打开项目，选择根目录的`pom.xml`，等待 Maven 自动下载依赖

3. 配置本地 Tomcat（10.1+/11.x 版本），添加项目 WAR 包进行部署

4. 启动 Tomcat，访问地址：`http://localhost:8080/discuss2`

### 4.2 服务器部署运行

1. 本地打包生成 WAR 包
        `mvn clean compile package`

2. 将 `target/discuss2.war` 上传至服务器 Tomcat 的`webapps` 目录

3. 启动 Tomcat 服务（先停止旧实例，避免冲突）
        `# 停止服务
/root/apache-tomcat-11.0.14/bin/shutdown.sh
# 启动服务
/root/apache-tomcat-11.0.14/bin/startup.sh`

4. 访问地址：`http://你的服务器IP:8080/discuss2`

## 六、注意事项

### 6.1 版本兼容

- JDK 版本必须统一：本地编译与服务器运行均使用 JDK 21，否则会出现类版本不兼容错误

- Tomcat 版本限制：仅支持 10.1.x 及以上版本，旧版本不兼容 Spring 6 的 Jakarta 规范

### 6.2 部署规范

- 端口配置：服务器需开放 8080 端口，或修改 Tomcat 配置文件自定义端口

- 缓存清理：上传 WAR 包前，删除 Tomcat `webapps` 目录下旧版项目文件及解压目录，避免缓存冲突

### 6.3 安全提示

- 敏感信息：请勿上传包含密码、密钥、服务器地址的配置文件

- 安全加固：服务器部署后，建议修改 Tomcat 默认端口与管理账号，提升项目安全性
